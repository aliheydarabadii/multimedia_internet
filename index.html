PART 2, SECOND SLIDES

Transport Protocols for Multimedia: TCP vs UDP

This section explains how multimedia applications, like video streaming, interact with the transport and network layers of the Internet.

The Setup

Imagine a Video Server and a Video Player communicating across the Internet. Between them lies a network cloud (the Internet), which operates on an IP Best-Effort basis. This means:

The network does its best to deliver data, but it doesn't guarantee delivery time, reliability, or correct order.

Protocol Layers in Action

Each application (video player and server) uses protocol layers:

  • Transport Layer Protocols: Responsible for managing data transfer.
    • TCP (Transmission Control Protocol)
    • UDP (User Datagram Protocol)

TCP vs UDP — What's the Difference?

TCP

  • Guarantees packet delivery.
  • Ensures the correct order of packets.
  • Slower, but reliable — ideal for applications needing accuracy.

UDP

  • Faster and lightweight.
  • Does not guarantee delivery or order — ideal for real-time streaming where speed is critical and small losses are tolerable.

Summary

"TCP guarantees packet delivery and correct sequence. UDP does not."

When developing real-time video or audio applications, UDP is preferred for speed, while TCP is used when reliability matters more than delay.

IP Best-Effort Service – Guarantees with TCP vs UDP

In an IP-based network, data is transmitted using a Best-Effort Service. This means the network tries its best to deliver packets but offers no strict guarantees on:

  • Delivery time (delay)
  • Packet delivery itself
  • Maintaining the correct packet order

To improve reliability, we use Transport Layer protocols like TCP or UDP over IP. However, their capabilities differ:

Feature TCP over IP UDP over IP
On Packet Sequence ✅ OK ❌ Still not guaranteed
On Packet Delivery ✅ OK ❌ Still not guaranteed
On Delay ❌ Still not guaranteed ❌ Still not guaranteed

TCP Over IP:

  • Ensures packets arrive in order.
  • Guarantees delivery of data.
  • Still does not guarantee delay, as delay is not managed at the transport layer.

UDP Over IP:

  • Does not guarantee delivery, packet order, or delay.
  • It's lightweight and faster, often used in real-time applications like voice and video, where speed matters more than reliability.

This table clearly shows that while TCP helps improve delivery and sequencing, both TCP and UDP still rely on IP's best-effort nature — meaning network delays remain unpredictable.

QoS and SLA: Moving Beyond IP for Better Multimedia Support

Multimedia applications such as video streaming demand Quality of Service (QoS) that the basic IP service alone cannot guarantee. The solution is to provide QoS at a higher layer, especially when handling real-time, high-quality streaming.

What is QoS?

Quality of Service (QoS) refers to a network's ability to guarantee performance metrics like:

  • Low latency
  • Minimal packet loss
  • Consistent throughput

These are especially important for video, voice, and interactive apps.

What is an SLA?

To formalize QoS expectations, networks use Service Level Agreements (SLAs). An SLA:

  • Defines a set of quality metrics (like availability or delay).
  • Represents a commitment from the service provider to meet specific performance targets.

Example – Traditional SLA Metric: Availability

One key QoS metric is Availability (A), which refers to how often a service or interface is up and running.

✅ Example SLA: The network interface must be available 99.975% of the time.

This means the defined equipment or service should be functional nearly all the time, ensuring consistent quality for critical applications.

Summary

The IP layer alone isn't enough for modern multimedia. By defining QoS via SLAs, providers can offer predictable, reliable service levels—a necessity for today's streaming and interactive applications.
